🔐 Secure Vault

A local, encrypted, offline-first password vault. No cloud, no accounts, no telemetry.

🗺Overview

All vault data lives in JSON files on your machine, encrypted with AES-256-GCM using a key derived from your master password via PBKDF2-SHA256 (600,000 rounds). The server component runs locally at http://localhost:7474 and shuts down when you close the app.

Key concepts

  • Vault — an encrypted container. You can have multiple vaults (personal, work, shared with team).
  • Entry — a single credential (login or secure note) stored inside a vault.
  • Session token — issued on unlock; required for every API call. Expires when the vault auto-locks.
  • Master password — never stored anywhere. If lost, the vault cannot be recovered.
Important: There is no password recovery. If you forget your master password the vault data is permanently inaccessible. Use 2FA backup codes as an additional safety net, not a password substitute.

✨Creating Your First Vault

  1. Launch Vault.exe. On first run, the unlock screen shows an Add Vault prompt.
  2. Click + Add Vault. Choose a label (e.g. "Personal") and a strong master password. Confirm the password.
  3. Optionally select a vault type: Local (default — file stored next to the exe) or Shared (file at a custom path, for network drives or shared folders).
  4. Click Create Vault. The success screen appears.
  5. Optionally configure Two-Factor Authentication now, or skip and do it later in Settings.

2FA during vault creation

After a vault is created you can pick any of four methods:

OptionWhat you need
📱 Authenticator AppGoogle Authenticator, Authy, or any TOTP app
📜 Backup CodesDownload 10 one-time recovery codes — store them safely
💌 Email OTPA Gmail (or other) account with SMTP app password configured
🔑 Security KeyYubiKey, Windows Hello, Touch ID, or any FIDO2 device

You can skip 2FA now and enable it any time from Settings → Security.

🏠Home Screen

The unlock screen is what you see when the app starts or the vault auto-locks.

  • Vault dropdown — select which vault to unlock. Only shows when at least one vault exists.
  • Edit Vault — button below the dropdown. Opens the vault edit modal to rename or remove the selected vault.
  • Master password field — type your password and press Enter or click Unlock.
  • PIN field — shown instead of the password field after you set up a PIN. Appears after the vault auto-locks mid-session. Use "Enter password instead" to fall back to the master password.
  • 2FA field — appears after correct password entry if 2FA is enabled on the vault.
  • + Add Vault — always visible; creates an additional vault alongside existing ones.
Multiple vaults: You can switch between unlocked vaults at any time using the dropdown. Each vault maintains its own session independently.

  2. Pick a template (or Blank) — this pre-fills the category and placeholder hints for common credential types.
  3. Fill in the fields. Use the password generator (⚡) to create a strong password.

📝Secure Notes

Select the Secure Note type when adding or editing an entry. Use this for:

The entry type is persisted — secure notes display with a note icon in the list view.

⚡Password Generator

Click the lightning bolt (⚡) next to the password field to open the generator. Configure:

  • Length — drag the slider (8–128 characters).
  • Character sets — toggle uppercase, lowercase, numbers, symbols.
  • Strength meter — real-time entropy estimate shown below the output.

Click the generated password to copy it, or click Use This to fill it into the entry form.

❤️Password Health

Available in the sidebar or via the toolbar icon. The health panel scans all entries in the current vault and flags:

  • Weak passwords — entropy below threshold.
  • Reused passwords — same password used in more than one entry.
  • Old passwords — entries with an expiry date that has passed.

Click any flagged entry to jump directly to it.

📄Password History

Every time you change an entry's password the previous one is saved automatically. The last 5 passwords are retained per entry.

In the entry detail panel, look for Password History below the Password field. Click Show N previous passwords to expand the list. Each historical password has a copy button and a reveal button.

  • History only records changes made while the vault is unlocked — the original password set on creation is not stored separately.
  • Use history to recover a credential if a site forced a rotation and you need the old value.
  • History entries cannot be deleted individually — they rotate out automatically as new changes push older ones off the 5-entry limit.

🔒Two-Factor Authentication

Open Settings → Security. The 2FA section shows the current status. If no 2FA is configured, click Enable 2FA. If 2FA is already active, click Edit 2FA to add more methods or disable existing ones.

Authenticator App (TOTP) — Recommended

  1. Click Enable 2FA → Authenticator App.
  2. Scan the QR code with Google Authenticator, Authy, 1Password, or any TOTP app. Alternatively, copy the secret key and enter it manually.
  3. Enter the 6-digit code from your app to verify, then click Verify & Enable.

Backup Codes

  1. Click Enable 2FA → Backup Codes.
  2. Download or copy all 10 codes. Each code is single-use.
  3. Check the "I have saved these codes" checkbox and click Done.

Once any 2FA is enabled, you can view or regenerate backup codes any time via Edit 2FA → Backup Codes.

Store backup codes offline. If you lose your authenticator and have no backup codes, you cannot unlock the vault. Print them or store in a secondary secure location.

Email OTP

  1. Click Enable 2FA → Email OTP.
  2. If SMTP is not yet configured, the SMTP setup dialog appears automatically. Use Gmail with an app-specific password.
  3. Enter the email address where codes should be sent. Click Send Test & Enable.
  4. Enter the code you receive in email to confirm.
Gmail setup: Go to myaccount.google.com → Security → 2-Step Verification → App passwords. Create a password for "Mail". Use that 16-character code — not your Google account password — in the SMTP App Password field.

Security Key (WebAuthn)

  1. Click Enable 2FA → Security Key.
  2. Click Register Security Key and follow your browser/OS prompt to tap your key, use Windows Hello, or use Touch ID.
  3. Once registered, future vault unlocks prompt you to tap your key after entering the master password.

Supported devices: YubiKey (any series), Windows Hello (PIN, fingerprint, face), Touch ID on Mac, any FIDO2/WebAuthn authenticator.

Multiple methods: You can enable more than one 2FA method on the same vault. At unlock, the vault will prompt whichever method is registered. If you have a security key registered, it takes priority.

Managing 2FA (Edit mode)

Click Edit 2FA in Settings → Security. The type picker opens with status badges on each tile — ✓ Enabled (green) or "Not configured" (gray). Click an enabled method to disable it; click a disabled method to set it up.

🞲PIN Quick-Unlock

After setting up a PIN, when the vault auto-locks during a session the unlock screen shows a PIN input instead of the master password field — saving you from retyping a long password.

  1. Go to Settings → Security → Quick-Unlock PIN → Manage PIN.
  2. Enter your master password to authenticate, then set a 6-digit PIN.
  3. When the vault auto-locks, type the PIN. The vault re-opens without a full unlock.
  • The PIN is only valid for the current app session. Closing and reopening the app clears it.
  • Manually locking the vault (lock button) also clears the PIN session — you must re-enter the master password.
  • Use Enter password instead on the unlock screen to bypass PIN and use the master password.
PIN security: The PIN is encrypted with AES-GCM + PBKDF2 in localStorage, not stored in plaintext. Still, choose a PIN you do not use elsewhere.

⏱Auto-Lock

The vault automatically locks after a configurable period of inactivity. Configure the timeout in Settings → Security → Auto-Lock Timeout.

  • Options: 1 min, 5 min, 15 min, 30 min, 1 hr, Never.
  • A yellow warning banner appears at the top of the app when 30 seconds remain: "Vault locking in 30s — Stay Active".
  • Click Stay Active or perform any action (click, keypress, mouse move) to reset the timer.
  • When the vault locks, unsaved form data is discarded. Save entries before stepping away.

🗃Local vs Shared Vaults

Local vault

The default. The vault file (<label>.json) is stored in the same directory as Vault.exe. Suitable for personal use on a single machine.

Shared vault

You specify a custom file path — a network share, NAS drive, or synced folder (Dropbox, OneDrive). Multiple users or machines can open the same vault file as long as each knows the master password. There is no conflict resolution; only one person should write at a time.

When you click + Add Vault → Shared, two options appear:

  • + Create New — Creates a brand-new encrypted vault file at the path you specify. Enter a vault name, the full file path (must end in .json), a new password, and confirm it. After creation you can optionally enable 2FA.
  • 🔗 Connect Existing — Registers a vault file that already exists (created by another machine or user). Enter the vault name, the path to the existing .json file, and its current password. The app verifies the password unlocks the file before adding it. No confirm-password field — no new vault is created. Once connected, the vault opens immediately.

💡 Use Create New when setting up a shared vault for the first time. Use Connect Existing on every other machine that needs access to the same file.

Vault file format

  • Plain JSON with encrypted payload — human-readable metadata (label, created date) and an AES-GCM-encrypted entries blob.
  • Files are safe to copy, backup, or move. A vault file opened on another machine with the same master password unlocks normally.

📦Managing Vaults

Switching vaults

Use the dropdown at the top of the unlock screen to select a vault. Unlocked vaults show [Unlocked] next to their name. You can have multiple vaults open simultaneously — switching between them does not lock the others.

Editing a vault

Select the vault in the dropdown and click Edit Vault. You can rename the vault or change its master password. If 2FA is enabled, you must provide a valid 2FA code before saving changes.

Removing a vault

Click Edit Vault → Remove Vault. Enter the master password to confirm. This removes the vault from the app's registry only — the .json file is not deleted from disk.

🔀Vault Merge

Combine two vaults into one. Go to Settings → Data → Merge Vault.

  1. Unlock the destination vault (the one you want entries merged into).
  2. Open Settings → Data and click Merge Vault.
  3. Select the source vault from the dropdown.
  4. Enter the source vault's master password.
  5. Click Merge. A summary shows how many entries were added and how many duplicates were skipped.

Deduplication logic

An entry is considered a duplicate if another entry in the destination vault has the same title and username (case-insensitive). Duplicates are skipped — the destination copy is kept unchanged.

Safe to re-run: Running a merge twice will not create duplicates. If you add new entries to the source vault later, merging again only imports the new ones.
The source vault is not modified. Merge is a read-only operation on the source. The source vault file remains intact after merging.

↕️Import & Export

Export

Go to Settings → Data → Export. Exports all entries from the current vault to a CSV file. The exported CSV is unencrypted — treat it as sensitive and delete it after use.

Import

Go to Settings → Data → Import. Supported format: CSV with columns title, username, password, url, notes, category. Extra columns are ignored.

To migrate from another secure vault: export it to CSV, ensure the column headers match (rename them if needed), then import.

📤Encrypted Files

Every vault can store files alongside passwords. Files are encrypted with the same AES-256-GCM key derived from your master password — they are never stored in plaintext on disk.

How It Works

Files are stored as entries inside the vault, the same way passwords are. When you upload a file:

  1. The file bytes are base64-encoded and added as an entry with entry_type: "file" in the vault JSON.
  2. The entire vault (including all file data) is encrypted with AES-256-GCM using the key derived from your master password.
  3. Nothing extra is written outside the vault file — no separate folders, no companion files.

This means backing up or moving your vault file automatically includes all uploaded files. Filenames, sizes, and contents are all encrypted together.

Uploading Files

  • Click Files in the left sidebar to open the file vault.
  • Drag & drop files onto the drop zone, or click + Upload Files to browse.
  • Multiple files can be uploaded at once. A progress indicator shows the current file.
  • There is no per-file size limit enforced by the application (constrained only by available disk space).

Downloading Files

Click ↙ Download on any file card. The server decrypts the file in memory and streams it to your browser, which triggers a standard save dialog. The decrypted bytes are never written to the .files/ folder.

Renaming & Deleting

  • ✎ Rename — updates the filename stored in vault metadata. The .enc file on disk keeps its UUID name.
  • 🗑 Delete — removes the metadata from the vault and deletes the .enc file from disk. This is permanent.

File Storage Location

For a vault stored at C:\Users\you\vault.json, encrypted files live at C:\Users\you\vault.files\<uuid>.enc. If you move or back up your vault, move the .files folder with it.

Good candidates for file storage: SSH keys, certificates, API tokens, seed phrases, important PDFs, scanned ID documents — anything sensitive you want encrypted at rest.

🛡️Best Practices

Vault password

  • Use a passphrase of 4–5 random words (e.g. correct-horse-battery-staple) — long, memorable, not guessable.
  • Do not reuse a vault password anywhere else.
  • Write it down and store it physically somewhere only you can access. There is no recovery option.

Two-factor authentication

  • Enable at least one 2FA method and generate backup codes.
  • Print backup codes and keep them with your physical password record — not on the same device.
  • A YubiKey or Windows Hello is the most phishing-resistant option; use it if available.

Entry passwords

  • Use the built-in generator. 20+ characters with mixed character sets is ideal for most sites.
  • Each site gets a unique password — never reuse. The health checker flags reused passwords.
  • Set expiry dates on sensitive accounts (banking, email) to prompt regular rotation.

Vault backups

  • Copy the vault .json file to an external drive or encrypted cloud storage periodically.
  • The vault file is safe to backup — it is encrypted at rest and cannot be read without the master password.
  • Consider using the Shared Vault type pointing to a synced folder for automatic off-machine copies.

🔧Troubleshooting

Blank screen on launch

The local server failed to start (port 7474 in use, or antivirus blocked it). Close any other instance of the app. If the problem persists, check Windows Firewall / antivirus exceptions for Vault.exe.

"Incorrect vault password"

The password is wrong, or the vault file was moved or corrupted. Verify the file path in the vault list (Settings → Data). If the file exists and you are sure the password is correct, the file may be corrupted — restore from a backup copy.

"Vault already exists, use unlock"

You tried to create a vault with a label that already exists. Choose a different label, or go to Settings → Data to remove the old entry first.

TOTP code rejected

TOTP codes are time-based. If your device clock is off by more than 30 seconds, codes will be invalid. Sync your system clock: Settings → Time & Language → Sync now. The app accepts ±1 time step (±30 s).

Email OTP not arriving

  • Check spam/junk folder.
  • Verify SMTP settings in Settings → Security → SMTP Configuration. Send a test email.
  • If using Gmail, ensure you used an app-specific password (not your account password). Regular passwords are rejected by Google's SMTP.
  • Amazon corporate (@amazon.com) accounts will never work — the tenant has SMTP AUTH disabled. Use a personal Gmail or Outlook account.

Security key not responding

  • The app must be accessed at http://localhost:7474 (not 127.0.0.1) for WebAuthn to work — this is enforced automatically.
  • If the browser prompt does not appear, try clicking "Register Security Key" / "Tap Security Key" again.
  • Windows Hello requires a PIN to be set in Windows Settings before it can be used as a security key.
  • YubiKey on USB: tap the key when the light starts blinking. NFC: hold it near the reader.

Import not working

Ensure your CSV has a header row with at least a title column. Other accepted column names: username, password, url, notes, category. Open the file in a text editor to verify the format.

PIN not appearing after auto-lock

The PIN session only persists within the same app run. If you closed and reopened the app, the PIN is cleared — enter your master password. Set the PIN again after unlocking to restore the quick-unlock for the current session.

Shared vault not detecting changes

The app polls the shared vault file every 5 seconds. If changes made by another user are not appearing, check that: both users are pointing to the same file path, the file system supports mtime timestamps (most do), and neither user's vault is locked (polling is paused when locked).

Vault merge shows 0 added entries

All entries in the source vault matched duplicates in the destination (same title and username). If you expected new entries to come through, check that they have distinct titles or usernames in the source vault before merging again.

Accent color or theme not saving

Settings are stored in browser localStorage. If you clear browser data or run the exe from a different user profile, stored preferences will reset to defaults. Re-apply your preferred color from Settings → Appearance.

Slow unlock

PBKDF2 with 600,000 rounds takes 0.5–2 seconds depending on hardware. This is intentional — it makes brute-force attacks impractical. If unlock consistently takes more than 5 seconds, your machine may have a hardware issue unrelated to this app.
